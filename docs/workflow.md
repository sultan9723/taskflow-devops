# Development Workflow

The TaskFlow project follows a structured Git and GitHub workflow to support collaborative development.

## Branches
- `main` stores the stable version of the project
- `develop` is used for integration of new work
- `feature/*` branches are used for individual feature development
- `bugfix/*` branches are used to resolve issues

## Process
1. Create a branch from `develop`
2. Implement changes with clear commits
3. Push the branch to GitHub
4. Open a pull request into `develop`
5. Address review comments before merging
6. Merge `develop` into `main` for stable release
